# ALT-PPT 사용 가이드 (대표님용)

> **한 줄 요약:** 코드를 짤 필요가 없습니다.
> **Claude Desktop → Code 모드**를 열고, 아래 문장을 **그대로 한국어로 말**하면
> Claude가 알아서 파일을 읽고, 디자인 시스템을 입히고, 슬라이드를 만들어 줍니다.

---

## 0. 준비 (딱 한 번만)

1. **Claude Desktop** 설치 → 로그인.
2. 상단/좌측에서 **Code(코드) 모드**로 들어갑니다.
3. 이 프로젝트 폴더(`ALT_PPT`)를 엽니다. (이미 GitHub에서 `pull` 하셨으면 그 폴더 그대로)
4. 준비 끝. 이제부터는 **채팅창에 한국어로 요청**만 하시면 됩니다.

> 💡 화면에 `python ...` 같은 명령어가 지나가도 **무시하셔도 됩니다.**
> 그건 Claude가 뒤에서 알아서 돌리는 것이고, 대표님이 직접 칠 필요는 없습니다.

이 프로젝트가 하는 일은 크게 3가지입니다.

| 상황 | 무엇을 하나 | 이 문서의 섹션 |
|---|---|---|
| 마음에 드는 PPT 디자인을 발견 | 그 디자인을 **우리 표준(디자인 시스템)** 으로 등록 | **A** |
| 이미 만든 PDF/PPTX가 있음 | 그 자료를 **우리 디자인으로 재생성** | **B** |
| 기획안(글)만 있음 | 기획안을 **슬라이드로 새로 생성** | **C** |

마지막 섹션 **D** 는 "이걸 Claude Design + Claude Code 만으로 하는 전체 흐름"입니다.

---

## A. 마음에 드는 디자인을 "디자인 시스템"으로 적용하기

좋은 PPT/PDF를 하나 발견했을 때, 그 **색·폰트·여백·레이아웃 규칙**을 뽑아
우리 표준으로 저장하는 단계입니다. 한 번 해두면 이후 모든 슬라이드가
이 규칙을 따라갑니다.

### 하는 법
1. 참고할 디자인 파일(PPTX 또는 PDF)을 이 폴더 안에 둡니다.
   (Keynote는 먼저 Keynote에서 **PDF로 내보내기** 하세요.)
2. Claude Code 채팅창에 이렇게 말합니다:

   > **"`OOO.pdf` 를 분석해서 우리 디자인 시스템으로 만들어줘."**

3. Claude가 색상(스포이드로 실제 픽셀 추출)·폰트·여백·슬라이드 레이아웃을 정리해
   아래 두 파일을 채웁니다.
   - `design-system/tokens.yaml` — 색/폰트/크기 등 **숫자로 된 규칙**
   - `design-system/claude-design-brief.md` — **사람이 읽는 브랜드 가이드** (이게 핵심)

> ✅ 지금 우리 프로젝트에는 **A'DREN 브랜드 덱** 기준으로 이미 만들어져 있습니다.
> (미니멀 흑백 에디토리얼 + 골드 포인트 1색) 새 브랜드로 바꿀 때만 이 단계를 다시 하면 됩니다.

### Claude Design에도 올려두기 (한 번만)
위 브랜드 가이드(`claude-design-brief.md`)의 내용을
[claude.ai/design](https://claude.ai/design) 의 **"Create here"** 에 붙여넣으면,
Claude Design이 그 뒤로 만드는 모든 시안이 이 브랜드를 따릅니다.
(우리는 이미 **"A'DREN / alternative nine — Deck Design System"** 으로 등록해 두었습니다.)

---

## B. 기존 PDF/PPTX를 우리 디자인으로 재생성하기 *(트랙 A)*

이미 있는 자료를 우리 표준 디자인으로 **다시 입히는** 작업입니다.

### 하는 법
1. 원본 파일(PPTX/PDF)을 이 폴더에 둡니다.
2. 채팅창에 말합니다:

   > **"`기존자료.pptx` 를 우리 디자인 시스템으로 재생성해줘."**

3. Claude가 자동으로:
   - 원본에서 **글·이미지·표**를 추출하고
   - 우리 브랜드 가이드를 입힌 **Claude Design용 프롬프트 + 이미지 묶음**을 만듭니다.
   - 결과 위치: `out/<이름>/claude-design-package/prompt.md` (+ `images/` 폴더)
4. 그 `prompt.md` 내용을 **Claude Design에 붙여넣고**, `images/` 폴더의 이미지를 첨부합니다.
5. Claude Design 캔버스에서 다듬은 뒤 **PDF로 내보내기(export)** → 최종본 완성.

> 💡 슬라이드 배치가 어색하면 "3번 슬라이드는 이미지 카드 형식으로 바꿔줘"처럼
> 말로 고치면 됩니다. 다시 자동으로 반영해 줍니다.

---

## C. 기획안(글)을 슬라이드로 새로 만들기 *(트랙 B)*

아직 슬라이드가 없고 **기획안·초안 글만 있을 때** 사용합니다.

### 하는 법
1. 기획안을 준비합니다. (텍스트 파일로 두거나, 채팅창에 그대로 붙여넣어도 됩니다.
   Notion·Google Drive에 있으면 그 위치를 알려주셔도 됩니다.)
2. 채팅창에 말합니다:

   > **"이 기획안을 우리 디자인 시스템으로 슬라이드 덱으로 만들어줘."**
   > (그리고 기획안 내용을 붙여넣기)

3. Claude가 자동으로:
   - 기획안을 **슬라이드 단위로 구조화**하고 (제목·핵심문장·레이아웃 지정)
   - 우리 브랜드를 입힌 **Claude Design용 프롬프트**를 만듭니다.
4. 그 프롬프트를 **Claude Design에 붙여넣어** 시안 생성 → 다듬기 → **PDF export**.

> 💡 "한 슬라이드에 한 메시지" 원칙으로 분량을 잡아줍니다.
> 표나 데이터가 있으면 자동으로 차트/표 레이아웃으로 배치합니다.

## D. Claude Design + Claude Code(Desktop) 만으로 하는 전체 흐름

별도 프로그램·터미널 지식 없이, **딱 두 개의 도구**만 씁니다.

```
┌─────────────────────────┐        ┌──────────────────────────┐
│  Claude Code (Desktop)   │        │  Claude Design           │
│  = 재료 준비 담당         │  ───▶  │  (claude.ai/design)      │
│                          │  붙여   │  = 실제 디자인·편집 담당  │
│ · 파일 읽기/추출          │  넣기   │                          │
│ · 브랜드 가이드 입히기     │        │ · 시안 생성/수정          │
│ · 프롬프트+이미지 생성     │        │ · 최종 PDF 내보내기       │
└─────────────────────────┘        └──────────────────────────┘
```

**역할 분담**
- **Claude Code(Desktop)** = 뒤에서 재료를 만드는 곳. 대표님은 한국어로 시키기만 하면 됨.
- **Claude Design** = 눈에 보이는 슬라이드를 실제로 만들고 고치고 **PDF로 뽑는** 곳.

**공통 순서 (A/B/C 모두 동일)**
1. **Claude Code**: "○○ 해줘" 라고 요청 → 프롬프트/이미지 묶음이 만들어짐.
2. **Claude Design**: 만들어진 프롬프트를 붙여넣고 이미지 첨부 → 시안 생성.
3. **Claude Design**: 말로 수정 지시 → 다듬기.
4. **Claude Design**: **PDF로 내보내기** = 최종 결과물.

### ⚠️ 꼭 기억할 3가지
1. **Claude Design은 브라우저 탭 안에서 그림을 그립니다.**
   생성 중에는 **그 탭을 닫지 마세요.** 닫으면 작업이 멈춥니다.
   (실수로 닫혔으면 "Resume" 버튼을 눌러 이어서 하면 됩니다.)
2. **첨부 파일은 용량 제한이 있습니다.** 너무 큰 원본(수십 MB)은 그대로 못 올라갈 수 있어,
   Claude Code가 슬라이드별로 잘게 나눈 `images/` 를 쓰는 게 안전합니다.
3. **최종 출력은 항상 PDF로 통일합니다.** (Claude Design의 export → PDF)

## 자주 쓰는 문장 모음 (복사해서 사용)

| 하고 싶은 것 | 채팅창에 붙여넣을 문장 |
|---|---|
| 새 디자인을 표준으로 등록 | `이 PDF를 분석해서 우리 디자인 시스템으로 만들어줘.` |
| 기존 자료 재생성 | `이 파일을 우리 디자인 시스템으로 재생성해줘.` |
| 기획안 → 슬라이드 | `이 기획안을 우리 디자인 시스템 슬라이드 덱으로 만들어줘.` |
| 특정 슬라이드 수정 | `3번 슬라이드를 이미지 카드 레이아웃으로 바꿔줘.` |

## 참고: 이 프로젝트 폴더 구성

```
design-system/     우리 브랜드 표준 (색·폰트·레이아웃 + 로고/참고 이미지)
  ├ tokens.yaml            숫자로 된 규칙
  ├ claude-design-brief.md 사람이 읽는 브랜드 가이드 ★핵심
  └ assets/                로고, 참고 썸네일
src/               파일 추출·프롬프트 생성 엔진 (Claude가 자동 실행)
out/               작업 결과물 (실행할 때마다 생성됨)
```

> 대표님은 이 폴더 구조를 외우실 필요가 없습니다.
> **"○○ 해줘"** 라고 말하면 Claude Code가 알아서 이 안에서 처리합니다.
